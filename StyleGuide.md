# Style Guide for Project

This is a styleguide for the project to ensure consistency 
in how I write the code and organise the project. 
<br>
<br>
### Directory Naming Scheme
Directories should be named in Pascal_Snake_Case. For example: Isolation_Forest.
<br>
<br>
### File Naming Scheme
Files should be named in PascalCase. For example, this file is called StyleGuide. If 
the file has only a single word in the name it should still be capitalised, e.g. Evaluation.py.
<br>
<br>
### Variable Naming Scheme
Variables should be named in camelCase. For example, thisVar. Constants should be SCREAMING_SNAKE_CASE. For example, a constant containing population count would be called something like POP_COUNT.
<br>
<br>
### Function Naming Scheme
Functions should be named in snake_case. For example a function name would be something like: import_directory().
<br>
